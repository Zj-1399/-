# Learn Command 实现细节

## 子命令详解

### 1. explore <主题>

**目的**：快速建立对陌生主题的整体认知

**实现步骤**：

```
1. 询问用户想达到的学习深度：
   - "想深入了解" → 探索 8-10 个核心概念
   - "快速入门" → 探索 4-5 个核心概念

2. 为每个核心概念提供：
   - 一句话定义
   - 与其他概念的关系
   - 学习优先级

3. 输出概念依赖图谱
```

**示例输出**：

```
# 机器学习 概念地图

## 核心概念（按学习顺序）

1. **监督学习** - 用标注数据训练模型
   → 是大多数 ML 应用的基础
   → 依赖：无

2. **无监督学习** - 从无标注数据中发现模式
   → 常用于数据聚类、降维
   → 依赖：监督学习基础

3. **特征工程** - 将原始数据转换为模型可用特征
   → 直接影响模型效果
   → 依赖：监督学习

...
```

### 2. study <主题> [--depth=N]

**目的**：渐进式学习单个概念

**实现步骤**：

```
1. 读取 learning-progress.json 获取已学习概念

2. 确定下一个应学习的概念（基于依赖关系）

3. 按 depth 级别提供内容：

   depth=1 (入门)：
   - 生活类比
   - 最简单的例子
   - 不超过 3 句话

   depth=2 (标准)：
   - 技术定义
   - 典型应用
   - 注意事项
   - 示例代码（如适用）

   depth=3 (深入)：
   - 数学/理论背景
   - 边界情况
   - 常见误区
   - 延伸阅读

4. 学习后更新 progress 文件
```

### 3. quiz <主题> [--count=N]

**目的**：通过主动回忆强化记忆

**问题类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| 定义题 | 解释概念 | "什么是过拟合？" |
| 关系题 | 说明关系 | "监督学习和无监督学习的区别？" |
| 应用题 | 场景选择 | "哪种情况适合用强化学习？" |
| 推理题 | 简单推导 | "如果增加模型复杂度会怎样？" |

**实现步骤**：

```
1. 读取 progress.json 获取薄弱概念

2. 生成 N 个问题（优先选择薄弱环节）

3. 逐个显示问题，等待用户回答

4. 评估回答（不要苛刻，重点是促进思考）

5. 记录 quiz 结果到 progress.json
```

### 4. track [主题]

**无主题**：显示所有学习主题概览

```
# 学习进度总览

| 主题 | 已学 | 掌握度 | 下次复习 |
|------|------|--------|----------|
| 机器学习 | 5/12 | 42% | 明天 |
| Python | 8/15 | 53% | 今天 |
```

**有主题**：显示特定主题详情

```
# 机器学习 详细进度

## 已掌握概念
- [x] 监督学习
- [x] 无监督学习
- [ ] 强化学习

## 需要复习
- [ ] 过拟合 (到期：昨天)

## 薄弱环节
- 梯度下降优化（正确率 60%）
```

### 5. teach <主题>

**目的**：通过讲解发现知识盲点

**实现步骤**：

```
1. 给出主题中的一个核心概念

2. 要求用户用自己的话解释

3. AI 评估并指出：
   - 正确的部分 ✓
   - 不清晰的部分 ?
   - 缺失的关键点 ×

4. 提供改进建议
```

**Feynman Technique 变体**：

```
概念：梯度下降

你的解释：[用户输入]

评估：
✓ 理解核心：沿着梯度方向优化
? 可以更清晰：学习率的作用解释不够
× 缺失要点：没有提到局部最优问题
× 缺失要点：没有区分 batch / SGD / mini-batch

建议：尝试用"下山找最低点"的比喻来说明学习率的作用
```

### 6. review

**目的**：间隔复习所有到期主题

**实现步骤**：

```
1. 读取所有主题的 next_review 日期

2. 找出所有到期（<= 今天）的概念

3. 按到期时间排序（先复习最久的）

4. 对每个概念进行快速测验

5. 根据表现更新下次复习时间
```

---

### 7. reset <主题>

**目的**：重置主题学习进度

**警告提示**：

```
⚠️ 确认要重置「机器学习」的学习进度吗？
   这将删除所有已记录的概念、测验历史和复习计划。

输入 "确认" 继续：
```

## 状态更新时机

| 操作 | 更新内容 |
|------|----------|
| study 完成 | 添加概念到 learned，更新 next_review |
| quiz 完成 | 更新 quiz_history，重新计算薄弱环节 |
| review 完成 | 更新 next_review（根据间隔重复算法） |

---

## 间隔重复算法

使用简化版 SM-2：

```
if quiz_score >= 0.8:
    next_interval = current_interval * 2
else if quiz_score >= 0.6:
    next_interval = current_interval * 1.5
else:
    next_interval = 1  # 重新开始

next_review = today + next_interval days
```
